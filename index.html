Comp 40

Machine Structure & Assembly Language Programming

Note: I cannot make this repository on Github public, due to Tufts policies. Please request access to it. If you have access, here is the link.

This class taught how machines store data and run programs. The structure of the machine itself was examined, including the CPU, caching, and computer memory. Representations of numbers in binary and hexidecimal were learned. Intel x86 Assembly Language was examined, along with the role of registers in computation. Big and Little-Endianness were looked at, as well as bitpacking and bit shifts.

This class also taught the importance of clean, modular code that worked under any circumstances. Testing and exception handling were stressed. All code had to free every byte of memory that was allocated. Abstraction and polymorphism were also stressed through void pointers and "unboxed" data structures.

These concepts were reinforced through the following eight projects, some of which used structures created by David Hanson and some of which used files provided by the instructors. All of these projects were completed with a partner:


  1. "filesnpix" - This project introduced us to the pnm file format for images. The first executable created calculated the average brightness of pixels in a greymap. The second, much more intensive executable, read in files and found all the similar lines between them, displaying them in a particular way.

  2. "iii" - This project introduced the concept of an "unboxed" data structure. We created our own unboxed, two-dimensional arrays. We used these arrays to take out the black edges from a bitmap, recursively. We also used arrays to calculate whether a given sudoku puzzle had been solved.

  3. "locality" - We learned about the concept of locality as it relates to caching. We created an abstracted 2D array - it could either be represented as an array of arrays or as an array of "blocks" of sequential memory. We used this structure to perform transformations on images - rotations, flips, etc. These transformations were timed to determine the best way to store the image.

  4. "arith" - Using a Discrete Cosine Transform and bitpacking, this image compressor/decompressor could take an image and reduce it to about 1/3 its original size. It could decompress the image back to its original, with some loss.

  5. "bomb" - A binary bomb had to be diffused by figuring out the correct inputs. This allowed us to look at moderately complex Assembly code. We also had to act as a decompiler and figure out the original C code, to a point.

  6. "um" - A 14-instruction virtual machine emulator was created in C. This machine reads and executes bitpacked instructions. It had segmented memory and 8 registers.

  7. "profiling" - Using profiling tools and optimization techniques, the Universal Machine was made as fast as possible. Ours increased in speed by about 95%.

  8. "calculator" - Using a macro for the Universal Machine, we simulated a fairly simple assmebly language. We used this language to create an RPN calculator with instructions that keeps track of numbers on a stack.
